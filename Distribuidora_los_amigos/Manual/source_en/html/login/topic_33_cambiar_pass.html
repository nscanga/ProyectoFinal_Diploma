Change password with token

Objective

This section describes the last step of the recovery process. Once the token is received by email, the user can set a new password by entering the code and the new access key.

Form fields

Field Description Type Required
Token Alphanumeric code sent by email. It expires once it is used. Text Yes
New password Password that replaces the previous one. It must have at least 6 characters. Secure text Yes
Confirm password Repeat the new password to avoid typing mistakes. Secure text Yes
Change password button Sends the request to RecuperoPassService.ChangePassword. Button ‚Äî

Procedure

  1. Enter the token received by email.
  2. Type the new password following the minimum length policy.
  3. Confirm the password by entering the same value.
  4. Press Change password. The button is disabled while the request is processed.
  5. When it completes successfully the message ‚ÄúPassword changed successfully‚Äù appears and the form closes.

Automatic validations

  • The token cannot be empty.
  • The password and its confirmation must match.
  • A minimum length of 6 characters is enforced.
  • If an error occurs it is logged by LoggerService.

Recommendations

Use passwords that combine letters, numbers, and symbols.
Do not reuse previous passwords or passwords used in other systems.
