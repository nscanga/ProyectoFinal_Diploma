System log

What is the log?

The log stores all relevant system actions: sign-ins, errors, maintenance tasks (backup/restore), and technical messages generated by LoggerService. This screen lets you review those records for audit purposes or incident resolution.

Entries are stored in the Bitacora database, table Log, and retrieved through BitacoraService.GetAllBitacora(), which reads the information via the SQL Server repository.

How to open it

  1. From the main screen open the menu assigned to the audit module (for example Backup & Restore).
  2. Select System log.
  3. The form displays the list of recorded events.
If you cannot see the option, ask an administrator to grant the required permissions or enable the screen for your role.

How to use the log

  1. Sort the grid (click the headers) to review the most recent events first.
  2. If the screen provides date or level filters, use them to focus on the period or severity you need to analyze.
  3. Select a row to read the complete message and, if present, the associated technical detail.
  4. Copy the relevant entries (Ctrl+C) and attach them to the incident documentation or support ticket.
The Ctrl+C shortcut copies the selected cells so you can paste them into emails or spreadsheets.

Understanding severity levels

  • Info: successful events or expected actions (user creation, completed backups).
  • Warning: unusual situations that did not stop the process but deserve follow-up.
  • Error: failures that prevented the operation from finishing. Check the related technical detail.
  • Critical/Fatal: severe infrastructure errors. Escalate immediately to the technical team.

Best practices

  • Review the log after each backup or restore process.
  • Keep evidence (screenshots or exports) when you detect recurring errors.
  • Limit access to users with audit roles or system administrators.
  • Establish a periodic review procedure (daily/weekly) to detect anomalies early.

Using the log to solve issues

Scenario What to look for Recommended action
Recent backup failed Error entries related to backup and the exception details. Validate folder/SQL permissions and retry the process once the cause is corrected.
Users report unexpected shutdowns Unhandled Exception errors including stack trace information. Forward to the development team including the Id_Log and the technical detail.
Access audit Info entries with sign-in/sign-out messages. Export the filtered list by user and date range to document the findings.
