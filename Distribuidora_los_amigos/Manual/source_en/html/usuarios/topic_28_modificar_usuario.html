Manage existing users

Objective

Use this screen to enable or disable existing users and review their current status. It is useful for handling temporary access or blocking accounts that should not be used.

How to access

  1. Select Management ‚Üí Edit user on the main screen.
  2. The form automatically loads the list of available users.

Main controls

Control Description Type Required
cbUsuarios User list loaded with UserService.GetAllUsuarios(). ComboBox Yes
Enable Button to enable a previously disabled user. Button ‚Äî
Disable Button that calls UserService.DisableUser to block access. Button ‚Äî

Procedure

  1. Select the user from the combo box.
  2. To block them, press Disable and confirm the operation.
  3. To enable them again, press Enable. (The action is logged even if enabling is pending implementation.)
  4. Check the informational messages and close the form when you are done.

Validations

  • You must select a user before executing any action.
  • Operations show a confirmation dialog to avoid accidental changes.
  • SQL errors are processed through ErrorHandler and descriptive messages are displayed.
  • All actions are logged with the user who performs them.

Recommendations

Use this screen together with View users to monitor the overall status.
Before disabling a user, confirm they do not have pending orders or critical tasks.
