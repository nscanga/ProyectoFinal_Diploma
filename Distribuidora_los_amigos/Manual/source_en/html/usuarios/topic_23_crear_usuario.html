Create user

What is it for?

Registers new users in the system. Each user receives login credentials and the basic login patent. Roles (families) and specific patents can be associated from other screens afterward.

How to access

  1. On the main screen go to Management ‚Üí Create user.
  2. The modal CrearUsuarioForm opens.

Field description

Field Description Type Required
User Unique name that identifies the user. Text (max. 50 characters) Yes
Password Initial password for the user. It is stored encrypted. Secure text Yes
Email Email address used for notifications and password recovery. Text No (recommended)
Create Button that executes UserService.Register and logs the operation. Button ‚Äî

Procedure

  1. Enter the username (without spaces or unusual special characters).
  2. Provide a temporary password following the internal security policies.
  3. Register a valid email to enable token-based recovery.
  4. Press Create. The system displays a confirmation message.
  5. Assign roles to the new user from the Assign role screen.

System validations

  • Username and password are required. A translated message appears if they are missing.
  • The service checks that no other user exists with the same name. Duplicates show the corresponding SQL error.
  • Exceptions are logged through LoggerService.

Tips

Follow a convention for usernames (for example, initial + last name).
Inform the user of the temporary password and ask them to change it on the first login.
