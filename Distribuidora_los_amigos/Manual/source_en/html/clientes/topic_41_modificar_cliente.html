Modify Customer

Introduction

This window is used to update the information of an existing customer. All the data is pre-loaded from the selected row in the customer list so you can adjust contact details, address, tax ID and activation status.

Keep customer records up to date to ensure the correct delivery of orders and accurate billing information.

How to Access

To access this feature:

  1. Open Customers ‚Üí View customers from the main menu.
  2. Select a customer in the grid and click Modify.
  3. The ModifyCustomerForm opens with the current data ready to edit.

Field Description

The table below describes each field in the form:

Field Description Type Required
Name Customer or company name. Text (max. 100 characters) Yes
Address Physical address used for deliveries. Text (max. 200 characters) Yes
Email Contact email. Must contain the character @ and not be a numeric string. Email Yes
Phone Contact phone number. Must contain only digits (no @). Numeric/Text Yes
CUIT / Tax ID Unique tax identification number for invoicing. Numeric (11 digits) Yes
Active Marks whether the customer can place new orders. Uncheck to temporarily deactivate the account. Checkbox Yes
Save Button that persists the changes and shows the confirmation message. Button ‚Äî

Procedure

1 Review the current information

Verify that the form corresponds to the customer you selected before applying changes.

2 Update contact data

Adjust the address, email and phone number. Make sure the email has a valid format and the phone contains only digits.

3 Confirm activity status

Use the Active checkbox to enable or disable the customer according to their current relationship with the company.

4 Save the changes

Click Save. If validation succeeds, the service updates the record and displays ‚ÄúCustomer successfully updated‚Äù.

System Validations

The system applies the following business rules before saving:

  • All fields (Name, Address, Email, Phone and CUIT) are mandatory.
  • The email cannot be only numbers and must contain @.
  • The phone must not contain the @ character.
  • If any rule fails, the form shows the detailed error message returned by ClienteService.ValidarCliente.

Tips and Recommendations

Double-check the CUIT before saving; it is used as the unique identifier for financial processes.
Use the F1 key to open the contextual help and review the validation criteria.
